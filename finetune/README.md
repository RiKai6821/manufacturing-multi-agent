# 模型微调指南（阿里云百炼 LoRA）

对应 JD：「调用国内大模型API，结合业务需求微调模型；Prompt工程及微调技术(LoRA/P-tuning)」

**关键优势：百炼平台微调无需自备 GPU**，在控制台上传数据即可训练，对个人学习极友好。

---

## 一、为什么微调（面试核心问答：微调 vs RAG vs Prompt）

| 手段 | 适用场景 | 本项目用在哪 |
|------|---------|------------|
| **Prompt工程** | 通用能力、轻量约束 | system提示、防幻觉约束 |
| **RAG** | 知识频繁更新、需可溯源 | 知识库检索（SOP/案例/规范） |
| **微调** | 固化"领域思维范式/输出风格" | 让模型养成诊断范式、稳定输出结构化报告 |

**一句话**：知识用 RAG（会变、要溯源），能力/风格用微调（固化诊断思路），约束用 Prompt。
三者不是替代关系，是配合关系——这正是本系统的设计。

## 二、数据集

运行 `python generate_dataset.py` 生成 `sft_dataset.jsonl`：
- **来源**：项目自有知识库（历史案例库 + SOP + 工艺规范）
- **格式**：百炼 SFT 标准 JSONL（system/user/assistant 三元组）
- **内容**：症状→根因诊断样本 + 领域知识问答 + 结构化诊断范式
- **数量**：35 条（可通过扩充知识库或增加问答模板继续扩大）

样本示例：
```json
{"messages": [
  {"role": "system", "content": "你是半导体良率诊断专家..."},
  {"role": "user", "content": "设备情况：良率骤降至88%，颗粒138超标，保养超期15天..."},
  {"role": "assistant", "content": "【根因分析】保养超期导致腔体污染...【处理建议】立即停机清洗..."}
]}
```

## 三、在百炼控制台微调（操作步骤）

```
1. 登录 bailian.console.aliyun.com
2. 左侧菜单「模型调优」→「创建训练任务」
3. 基础模型：选 qwen-plus / qwen-turbo（支持微调的版本）
4. 训练方式：选 LoRA（高效微调，训练快、成本低）
5. 上传数据集：sft_dataset.jsonl
6. 超参（用默认即可起步）：
   - learning_rate: 1e-4 左右
   - epochs: 3
   - LoRA rank: 8
7. 提交训练 → 等待完成（通常几十分钟）
8. 得到微调后模型ID，形如：qwen-plus-ft-xxxxxxxx
```

## 四、使用微调后的模型

微调完成后，只需改一处配置即可全系统切换：

```python
# settings.py
chat_model: str = "qwen-plus-ft-xxxxxxxx"   # 换成你的微调模型ID
```

其余代码一行不用动——这就是配置中心化的价值。

验证脚本：
```bash
python use_finetuned.py   # 对比微调前后模型在诊断任务上的输出差异
```

## 五、LoRA 原理（面试要点）

```
全量微调：更新模型全部参数 → 慢、贵、需大显存
LoRA：冻结原模型，只训练插入的低秩矩阵(A×B) → 参数量降到0.1%~1%
      训练快、显存小、可热插拔，效果接近全量微调
P-tuning：只训练连续的提示向量(soft prompt)，参数更少，适合轻量任务
```

为什么选 LoRA：在保持原模型通用能力的同时，低成本注入领域能力，是工业界微调的主流方案。
